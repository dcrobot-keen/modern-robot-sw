
## 로봇이란 무엇인가?

로봇은 "복잡한 일련의 동작을 자동으로 수행할 수 있는 기계, 특히 컴퓨터로 프로그래밍 가능한 기계"로 정의됩니다. 로봇을 구분하는 핵심 요소는 다음과 같습니다:

- **복잡한 동작의 자동 실행**
- **유연성을 위한 컴퓨터 프로그래밍 가능성**
- **환경에 적응하기 위한 센서 사용** (종종 간과되지만 매우 중요한 요소)

## 로봇의 분류

### 환경과 이동성에 따른 분류

**고정형 로봇**

- 제어된 환경에서 동작하는 산업용 로봇 매니퓰레이터
- 조립 라인, 수술용 로봇 등
- 안정적인 작업 환경

**이동형 로봇**

- 불확실한 환경을 탐색
- 예: 로봇 청소기, 자율주행차
- 세부 분류:
    - 수중 로봇 (해저 탐사)
    - 지상 로봇 (바퀴형, 다리형, 무한궤도형)
    - 공중 로봇 (드론, 비행선, 항공기)

### 응용 분야에 따른 분류

- **산업용**: 제조, 생산
- **서비스용**: 가정, 운송, 국방
- **의료용**: 수술, 재활, 훈련
- **교육용**: 학습 도구

## 로봇의 주요 유형

### 산업용 로봇

산업용 로봇은 잘 정의된 환경 덕분에 가장 먼저 개발되었습니다.

**특징**

- 반복적인 작업 수행 (용접, 도색)
- 점차 인간과의 협업 증가
- 안전을 위한 속도 제한 및 기계적 설계 필요

**장점**

- 로봇과 인간이 각자 잘하는 일을 수행
- 로봇: 반복적이거나 위험한 작업
- 인간: 복잡한 단계 및 전체 작업 관리

### 자율 이동 로봇

이동 로봇의 자율성 수준:

**원격 조종**

- 조작자가 제어
- 파이프 검사, 폭탄 제거 등
- 센서로 원격지 접근 제공

**반자율**

- 하위 작업 자동화
- 예: 드론 자동 비행 안정화 + 인간의 경로 선택

**완전 자율**

- 독립적 의사결정 및 탐색
- 불확실한 지형 및 변화하는 환경에서 작동

**진화 과정**

1. 초기: 단순한 환경 (수영장 청소, 잔디 깎기)
2. 현재: 실내 환경 탐색 (로봇 청소기)
3. 진행 중: 복잡한 환경 (자율주행차, 화성 탐사 로버)

**화성 탐사 로버**

- Sojourner: 1997년 3개월 활동
- Curiosity: 2012년 착륙 이후 현재까지 활동
- 반자율 시스템 (지구에서 임무 제어 + 자율 위험 회피)

### 휴머노이드 로봇

인간 형태의 로봇은 공상과학과 대중매체에서 인기가 많습니다.

**역사**

- 18세기 스위스 시계 제작자들의 휴머노이드 오토마타
- 현대 기업들도 홍보 목적으로 제작

**용도**

- 보행 역학 연구
- 인간-기계 상호작용 연구
- 가정 서비스 제안
- 노인 돌봄 (친근감 제공)

**과제**

- 매우 복잡한 기계 설계
- 다관절 제어의 어려움
- 높은 제작 비용
- "언캐니 밸리" 현상 가능
- 대부분의 용도에는 바퀴형이 더 효율적

### 교육용 로봇

기술 발전으로 비교적 저렴한 로봇 제작이 가능해졌습니다.

#### 사전 조립형 이동 로봇

**예시**

- Thymio (Mobsya)
- Dash (Wonder Workshop)

**장점**

- 즉시 사용 가능
- 다양한 센서와 출력 장치 포함
- 기계 설계에 시간 투자 불필요
- 견고하고 저렴

**단점**

- 수정 제한적
- 일부는 LEGO® 부품으로 확장 가능

#### 로봇 키트

**LEGO® Mindstorms**

- 1998년 출시 (최신 버전: EV3, 2014년)
- 구성: 표준 브릭 + 모터 + 센서 + 프로그래밍 가능 브릭

**장점**

- 유연한 설계 가능
- 상상력에만 제한됨
- 기계 설계 학습 가능

**단점**

- 사전 조립형보다 비쌈
- 견고한 기계 설계 능력 필요

**3D 프린팅 키트**

- 예: Poppy Ergo Jr 로봇 팔
- 기계 구조의 유연성 증가
- 높은 견고성
- 3D 프린터 접근 필요

#### 로봇 팔

**액추에이터의 역할**

- 환경에 작용하는 로봇 구성요소
- 산업용: 그리퍼 등 엔드 이펙터
- 이동형: 모터, 진공 펌프 등

**교육용 로봇의 특성**

- 대부분 모터와 디스플레이만 사용
- 키트나 추가 부품으로 엔드 이펙터 제작 가능
- 교육용 로봇 팔도 존재 (예: Poppy Ergo Jr)
- 객체 조작은 설계 복잡도 증가

#### 소프트웨어 개발 환경

**프로그래밍 언어 옵션**

1. 표준 언어 (Java, Python)
2. 블록 기반 (Scratch, Blockly)
3. 완전 그래픽 (VPL, Wonder)

**VPL (Visual Programming Language)**

- Thymio 로봇용
- 이벤트-액션 쌍 사용
- 왼쪽: 이벤트 블록
- 오른쪽: 실행 액션

**Wonder**

- Dash 로봇용
- 노드로 액션 표현
- 화살표로 이벤트 표현

## 일반 로봇 사양

이 책에서 사용하는 일반 로봇의 사양입니다. 교육용 로봇과 유사한 기능을 가지고 있습니다.

### 차동 구동 시스템

**구조**

- 독립적으로 구동되는 두 개의 바퀴
- 모터 출력: -100(전속력 후진) ~ 0(정지) ~ 100(전속력 전진)
- 앞쪽 지지 바퀴 (안정성)

**특징**

- 모터 출력과 로봇 속도는 미리 정의된 관계 없음
- 기어비, 타이어 유형, 지형에 따라 달라짐

**제자리 회전**

- 두 바퀴 중심점을 축으로 회전
- 수직 축 기준

**장점**

- 단순한 설계 (모터 2개만 필요)
- 조향 메커니즘 불필요
- 제자리 회전 가능
- 복잡한 조작을 단순한 동작으로 수행

**단점**

- 세 번째 지지점 필요
- 측면 이동 불가 (회전 필요)
- 무한궤도는 높은 마찰력 발생

**비교: 자동차 (Ackermann 조향)**

- 2개 또는 4개 바퀴 구동
- 복잡한 조향 메커니즘
- 제자리 회전 불가
- 평행 주차 같은 복잡한 조작 필요

### 센서 시스템

#### 근접 센서

**기능**

- 로봇 근처 물체 감지
- 수평 방향 감지

**기술**

- 적외선, 레이저, 초음파 등

**측정값**

- 거리 (센티미터)
- 각도 (도)
- 예: 로봇 중심에서 3cm, 정면 기준 45°

**실제 제한사항**

- 교육용 로봇: 소수의 센서만 보유
- 모든 방향 감지 불가능
- 저렴한 센서: 제한된 거리, 낮은 정확도
- 환경 요인 영향 (물체 유형, 주변 조명)

#### 지상 센서

**위치**

- 로봇 바닥에 장착
- 전방에 2개 (일반적)

**측정**

- 지면 반사 밝기
- 0 (완전 어두움) ~ 100 (완전 밝음)
- 임의 단위

**특징**

- 지면과 매우 가까움
- 거리나 각도 측정 불가
- 밝기만 측정

### 내장 컴퓨터

**기능**

- 센서 값 읽기
- 모터 출력 설정
- 정보 표시 (화면, 색상 LED)
- 데이터 입력 (버튼, 키패드, 리모컨)

**이벤트 처리**

- 버튼 터치 등의 이벤트 발생 시 실행
- 이벤트 핸들러 실행
- 하드웨어 인터럽트 또는 소프트웨어 폴링

**이벤트 vs 순차 프로그램**

- 순차: 초기 입력 → 처리 → 최종 출력
- 이벤트: 예측 불가능한 이벤트에 반응
- 그래픽 UI, 스마트폰 앱에서 사용
- 센서 값이 임계값 초과/미만 시에도 발생

**타이머**

- 스마트폰 스톱워치와 유사
- 시간 기간 설정 (예: 0.5초 = 500ms)
- 하드웨어 클록이 값 감소
- 0이 되면 "만료" → 인터럽트 발생

**타이머 용도**

- 반복 이벤트 (LED 깜빡임)
- 폴링 (주기적으로 센서 읽기 및 저장)

## 알고리즘 형식

로봇 제어에 사용되는 알고리즘은 의사코드(pseudocode)로 표현됩니다.

**의사코드 특징**

- 특정 프로그래밍 언어에 독립적
- 자연어 + 수학 + 프로그래밍 구조 조합
- Python처럼 들여쓰기로 범위 표시
- 할당에 화살표(←) 사용
- 등호(=)와 부등호(≠)는 수학 공식용

**예시: 정수 곱셈 알고리즘**

```
integer x ← 0
integer a, b
1: a ← 정수 입력
2: b ← 음이 아닌 정수 입력
3: while b ≠ 0
4:   x ← x + a    // a를 x에 더함
5:   b ← b - 1    // b번 반복
```

**모터 제어**

```
left-motor-power ← 50
right-motor-power ← -50
```

**센서 조건 (자연어)**

```
when object detected in front
when object not detected in back
```

## 책의 구성

### 핵심 챕터 (1-6장)

**1장: 로봇과 응용 분야**

- 로봇 분류
- 일반 로봇 사양
- 알고리즘 형식

**2장: 센서**

- 자율 동작의 핵심
- 범위, 해상도, 정밀도, 정확도
- 비선형성 처리

**3장: 반응형 동작**

- 센서 입력에 따른 직접 반응
- Braitenberg 차량
- 라인 따라가기 알고리즘

**4장: 유한 상태 머신**

- 상태별 다른 반응
- 상태 전환 형식화
- 이벤트 기반 전환

**5장: 로봇 운동과 오도메트리**

- 운동 개념 (거리, 시간, 속도, 가속도)
- 오도메트리: 위치 이동의 기본 방법
- 오차의 특성과 이해
- 고급 개념:
    - 휠 엔코더
    - 관성 항법 시스템
    - 자유도와 홀로노미

**6장: 제어**

- 폐쇄 루프 제어 시스템
- 센서 → 행동 → 센서 (순환)
- 최적 동작 보장 (예: 신호등 앞 부드러운 정지)

### 네비게이션 챕터 (7-10장)

자율 이동 로봇의 핵심 과제: 시작 위치에서 목표 위치로의 이동

**7장: 국소 네비게이션 - 장애물 회피**

- 기본 요구사항: 충돌 방지
- 로봇 주변의 즉각적 환경 처리
- 벽 따라가기 알고리즘
- 미로 탐색 알고리즘
- 확률적 알고리즘 (개미 집단 시뮬레이션)

**8장: 위치 추정**

- GPS 이전의 종이 지도 네비게이션
- 지도에서 현재 위치 파악 문제
- 시각 없이 해결해야 하는 과제
- 알려진 위치에서 삼각 측량
- 확률적 위치 추정:
    - 랜드마크 감지
    - 유사한 랜드마크가 여러 개일 경우
    - 확률 할당 및 업데이트
    - 이동하며 위치 확정

**9장: 지도 작성**

- 지도의 출처 문제
- 거리 지도: 쉽게 구할 수 있음
- 실내 지도: 로봇 청소기는 아파트 지도 없음
- 미지 환경: 해저 로봇 탐사
- 딜레마:
    - 위치 추정 → 지도 필요
    - 지도 작성 → 위치 추정 필요
- 해결: SLAM (동시 위치 추정 및 지도 작성)
- 환경 탐색 알고리즘
- 간소화된 SLAM 알고리즘

**10장: 지도 기반 네비게이션**

- 시작→목표 경로 계획
- Dijkstra 알고리즘 (최단 경로)
- A* 알고리즘 (휴리스틱 활용한 효율적 버전)

### 고급 주제 (11-16장)

서로 독립적이므로 선택적으로 학습 가능

**11장: 퍼지 논리 제어**

- 기존: 정확한 목표값 필요 (예: 온도 22°C, 속도 80km/h)
- 퍼지: 모호한 명세 사용
    - 온도: 춥다, 시원하다, 따뜻하다, 뜨겁다
    - 속도: 매우 느리다, 느리다, 빠르다, 매우 빠르다
- 물체 접근 로봇 제어 예시

**12장: 영상 처리**

- 대부분의 센서: 거리/각도 측정 (레이저, 음파, 적외선)
- 인간: 주로 시각 의존
- 디지털 카메라: 저렴하고 보편화 (스마트폰)
- 과제: 영상 처리 및 해석 (뇌는 즉시 처리)
- 계산 능력 필요
- 영상 처리 알고리즘 개요
- 카메라 없이도 교육용 로봇으로 시연 가능

**13장: 신경망**

- 복잡한 환경의 모든 상황에 대한 알고리즘 불가능
- 자율주행차: 모든 차량 구성 미리 알 수 없음
- 경험을 통한 학습 필요
- 인공지능의 기본 주제
- 인공 신경망: 뇌의 뉴런 모델
- 학습 알고리즘으로 내부 매개변수 수정
- 새로운 상황에 지속적 적응

**14장: 머신러닝**

- 통계 기법 기반 학습
- 두 가지 대안 구분 (예: 빨간 신호등 vs 녹색 신호등)
- 선형 판별 분석 (Linear Discriminant Analysis)
    - 샘플의 평균과 분산 기반
- 퍼셉트론
    - 신경망의 한 형태
    - 통계 가정 불필요
    - 더 유연한 구분 가능

**15장: 군집 로봇**

- 시스템 성능 향상 방법:

## 핵심 개념 정리

### 오토마타 vs 로봇

**오토마타**

- 기계적으로 프로그래밍
- 제한된 센서
- 유연성 낮음

**로봇**

- 컴퓨터 프로그래밍 가능
- 센서 기반 적응
- 높은 유연성

### 자율성 수준

**원격 조종**

- 조작자가 모든 것 제어
- 센서는 원격 접근용

**반자율**

- 일부 작업 자동화
- 인간이 주요 결정

**완전 자율**

- 독립적 의사결정
- 센서 기반 환경 이해

### 차동 구동의 특성

**장점**

- 설계 단순 (모터 2개)
- 제자리 회전 가능
- 정밀한 조작
- 복잡한 경로를 단순 동작으로

**단점**

- 3번째 지지점 필요
- 측면 이동 불가
- 무한궤도 사용 시 높은 마찰

**용도**

- 교육용 로봇 (대부분)
- 중장비
- 군용 탱크
